What it is

A requirement in, buildable designs out

A customer application goes in, described in plain terms: how long it has to run, how hard it has to pull, the space and weight it has to fit. The tool searches the manufacturable design space with a validated physics model and returns a short list of buildable cell designs, ranked.
  • It does the early design exploration: which part of the design space to even look at.
  • Engineers review the candidates, then build and test the best ones. That work does not change.
  • It competes with the first guess, not the lab. The goal is fewer build-and-test loops.

The DFN, implemented and validated

PyBaMM's DFN, validated against real LG M50 discharge data (Chen2020), at four discharge rates.
  • The physics shape is sound across every rate.
  • The main error is a near-constant offset, the kind calibration removes.

Where it stands

Built on research data. Ready for IBC's.

Everything so far runs on published cell data. The structure is the same one IBC would use; the values plug in by configuration, not a rewrite.
  • Plug in IBC's cell parameters so the designs are IBC's, not the published cell's.
  • Calibrate the DFN to IBC test data so the directional numbers become design-grade. This is the loop a vendor cannot copy, because the data stays inside IBC.
  • Each build and test feeds back, and the next design starts from everything learned before it.
